# 1.openclaw简介

OpenClaw 不是单纯“陪你聊天”的 AI，而是一个运行在你自己设备上的 AI 助手。  
普通大模型主要负责回答问题，OpenClaw 更进一步，它能连接聊天软件、浏览器和定时任务等工具，替你执行实际操作。它的核心是本地运行的 Gateway，相当于一个中枢，把聊天入口、模型能力和各种工具连接起来，让 AI 不只是会说，还能真正帮你做事。这也是它和web端通用大语言模型最大的区别：后者是语言对话，前者是由语言模型驱动的持续运行的代理系统。 

# 2.安装方式

可以完全参考ai，由于openclaw是macOS/linux应用，所以Mac端要方便一点，windows端要使用wsl，不过放心，在可以预见的短暂未来windows端一定会诞生类似应用（可能已经有了）。

# 3.核心创新点：

首次将ai的操作权下放到个人电脑终端，并用大语言模型生成操作指南。事实上用语言模型作为中介连接多模态输入输出端早已是空间智能确立的两条实现路径之一（另一条是基于输入及输出的模态进行针对性神经网络训练，不在中间插入语言模型），在工业场景里已经有成熟应用，如西门子 2025 年公开的 Industrial Copilot for Operations项目。openclaw的创新在于将工业场景移植到了个人终端，让普通个人用户也能尝试该应用。

# 4.openclaw的安全风险具体在哪？如何避免？

## 安全风险：

1. 权限过大：由于openclaw有在终端控制电脑的权力，所以Gateway 认证暴露、浏览器控制暴露、过高的 allowlist、文件系统权限过宽，都会放大风险。  
2. skills/插件风险：skills是openclaw生态的主要攻击面之一，GitHub issue 中甚至直接写到确认过的大规模 supply-chain attack。  
3. 提示注入与不可信输入：微软的安全分析提到一个很关键的点：这类自托管代理同时面临两条供应链风险——一条是“不可信代码”，另一条是“不可信指令”。也就是说，不只是装进去的 skill 可能有问题，外部网页、邮件、聊天消息、文档里的内容本身也可能被设计成诱导代理执行错误操作。因为 OpenClaw 这类系统会把“看到的文本”转进“可执行的动作链”，所以恶意输入可能从一段文字，变成一次真实操作。

## 如何避免：

1. 最小权限原则：不要开放所有终端平台的权限，而是针对你需要的任务开放对应部分的权限。  
2. 尽量少装第三方skills：必须装开源社区skills时先核来源、看扫描结果、看代码和 issue 记录。  
3. 只从官方渠道安装和更新。  
4. 在高风险操作上保留人工确认环节。  

# 5.openclaw代表的应用领域在未来是否有真正的发展价值？

我认为个人终端上的ai代理在未来一定会有非常非常显著的应用价值，并不是泡沫。它所期望解决的问题早已提出，如果你前几年对web3有所了解的话，你会知道有一个叫语义网的东西是当时所提出的下一代互联网未来概念，核心是让机器理解网络信息并在终端上进行操作，很明显当下openclaw所遵循的发展方向正是这个语义网当年许下的愿景。把它和 Web3、语义网放在一起看，就更容易理解为什么真正优秀的未来发展方向是可以预测的。语义网当年最核心的理想，是让网络上的数据不只是“可显示”，而且“可被机器理解与重用”。W3C 对 Semantic Web 的经典定义，就是提供一个通用框架，使数据能够跨应用、企业和社区边界被共享和复用；RDF 也是为此建立的 Web 数据交换标准。换句话说，语义网真正预言到的，不是某个具体产品，而是一个趋势：未来的网络一定会从面向人阅读，转向同时面向机器处理。尽管当下的相关应用还远远没有达到语义网的成熟实现，但可以确定的是当初领域专家们所定义的web3时代如今已经开始了，它正在我们脚下。

# 6.openclaw在当下对普通人来说是热炒的概念还是真正有用的工具？

虽然现在openclaw是最火热的概念，所有人都在匆忙进场，然而大部分普通人并没有流程化、标准化、自动化生产的需求，大部分需要ai协助的领域依旧需要大量人工指令微调。所以与其将openclaw当作“非用不可的工具”，不如当成一个“值得关注的新方向”。它就像早期的chatgpt甚至更早期的爱丽丝人机对话robot，虽然当下并没有广泛的应用空间，但它代表着一条正在成型的产品路线：在可以预见的未来会有越来越多的应用领域，它们能真正解决你的真实需求，而且它们马上到来。

# 总结：

Openclaw在中国的爆火，有技术方向上的革新，也有自上而下的推力，高涨情绪下所有人都在争先恐后入场，生怕落后就要挨打。前段时间看到腾讯总部楼下人山人海的openclaw线下排队安装活动，我瞬间诡异地感受到一生焦虑的东亚人的具象化群像，诡异的矛盾点在于我认为在底层思想上如果没有借助ai来指导生活生产的惯性的话，他们在当下大概率也不需要借助ai进行更上层的自动化生产，不过这至少体现了中华民族源于农耕文明历史的一项优秀品质——由于自己所拥有的土地资源是固定的，而且我们通常也不邪恶地侵略他人的土地，所以会将生产力提高的愿景寄托在生产工具的更新上。虽然ai革命的时代浪潮已经不可逆地到来了，但至少在openclaw这一领域上，你现在并不用过分焦虑，你几乎不会错过什么，只需要耐心等待应用市场上能真正解决你真实需求的工具出现。
